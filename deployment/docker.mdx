# Docker Deployment

Run Ello using Docker for consistent deployments across environments.

## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) installed
- [Docker Compose](https://docs.docker.com/compose/install/) (optional, for easier management)

## Quick Start

### Build the Image

```bash
docker build -t ello .
```

### Run the Container

```bash
docker run -d \
  --name ello \
  -p 3000:3000 \
  -v ello-data:/data \
  ello
```

Access the app at `http://localhost:3000`

## Docker Compose

Create a `docker-compose.yml` file:

```yaml
version: '3.8'

services:
  ello:
    build: .
    container_name: ello
    ports:
      - "3000:3000"
    volumes:
      - ello-data:/data
    environment:
      - NODE_ENV=production
      - DATABASE_PATH=/data/contacts.db
      - PHOTOS_PATH=/data/photos
      - PORT=3000
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 10s

volumes:
  ello-data:
```

### Start with Docker Compose

```bash
docker-compose up -d
```

### Stop

```bash
docker-compose down
```

### View Logs

```bash
docker-compose logs -f
```

## Dockerfile Explained

The project uses a multi-stage Dockerfile:

```dockerfile
# Stage 1: Build frontend
FROM node:20-alpine AS frontend-build
WORKDIR /app/frontend
COPY frontend/package*.json ./
RUN npm ci
COPY frontend/ ./
RUN npm run build

# Stage 2: Build backend
FROM node:20-alpine AS backend-build
WORKDIR /app/backend
COPY backend/package*.json ./
RUN npm ci
COPY backend/ ./
RUN npm run build

# Stage 3: Production
FROM node:20-alpine
WORKDIR /app

# Install production dependencies only
COPY backend/package*.json ./
RUN npm ci --only=production

# Copy built artifacts
COPY --from=backend-build /app/backend/dist ./dist
COPY --from=frontend-build /app/frontend/dist ./public

# Create data directories
RUN mkdir -p /data/photos

# Set environment
ENV NODE_ENV=production
ENV DATABASE_PATH=/data/contacts.db
ENV PHOTOS_PATH=/data/photos
ENV PORT=3000

EXPOSE 3000

CMD ["node", "dist/server.js"]
```

### Build Stages

| Stage | Purpose | Output |
|-------|---------|--------|
| `frontend-build` | Compile React app | `frontend/dist/` |
| `backend-build` | Compile TypeScript | `backend/dist/` |
| Production | Run the app | Final image |

### Image Size

The multi-stage build produces a small production image:
- Base: ~180MB (node:20-alpine)
- App: ~50MB
- **Total: ~230MB**

## Volume Management

### Named Volume

```bash
# Create volume
docker volume create ello-data

# List volumes
docker volume ls

# Inspect volume
docker volume inspect ello-data

# Remove volume (WARNING: deletes all data)
docker volume rm ello-data
```

### Bind Mount (Development)

Mount a local directory for easier access:

```bash
docker run -d \
  --name ello \
  -p 3000:3000 \
  -v $(pwd)/data:/data \
  ello
```

## Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `NODE_ENV` | `production` | Environment mode |
| `DATABASE_PATH` | `/data/contacts.db` | SQLite database path |
| `PHOTOS_PATH` | `/data/photos` | Photo storage path |
| `PORT` | `3000` | Server port |

### Custom Environment File

Create `.env.docker`:

```bash
NODE_ENV=production
DATABASE_PATH=/data/contacts.db
PHOTOS_PATH=/data/photos
PORT=3000
```

Run with env file:

```bash
docker run -d \
  --name ello \
  --env-file .env.docker \
  -p 3000:3000 \
  -v ello-data:/data \
  ello
```

## Health Check

The container includes a health check:

```bash
# Check container health
docker inspect --format='{{.State.Health.Status}}' ello

# View health check logs
docker inspect --format='{{json .State.Health}}' ello | jq
```

## Backup and Restore

### Backup Volume

```bash
# Create backup
docker run --rm \
  -v ello-data:/data \
  -v $(pwd):/backup \
  alpine tar czf /backup/ello-backup.tar.gz /data

# The backup is now at ./ello-backup.tar.gz
```

### Restore Volume

```bash
# Restore from backup
docker run --rm \
  -v ello-data:/data \
  -v $(pwd):/backup \
  alpine tar xzf /backup/ello-backup.tar.gz -C /
```

## Reverse Proxy

### Nginx Configuration

```nginx
server {
    listen 80;
    server_name contacts.yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### Traefik Labels

```yaml
services:
  ello:
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.ello.rule=Host(`contacts.yourdomain.com`)"
      - "traefik.http.routers.ello.tls=true"
      - "traefik.http.routers.ello.tls.certresolver=letsencrypt"
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Container won't start">
    Check logs:
    ```bash
    docker logs ello
    ```
    Common issues:
    - Port already in use
    - Volume permission errors
    - Missing environment variables
  </Accordion>

  <Accordion title="Database permission denied">
    Ensure the data directory is writable:
    ```bash
    docker exec ello ls -la /data
    ```
    If needed, fix permissions:
    ```bash
    docker exec ello chmod 755 /data
    ```
  </Accordion>

  <Accordion title="Photos not loading">
    Check the photos directory:
    ```bash
    docker exec ello ls -la /data/photos
    ```
    Ensure `PHOTOS_PATH` matches the volume mount.
  </Accordion>

  <Accordion title="Out of memory">
    Large imports may need more memory. Increase the limit:
    ```bash
    docker update --memory=1g ello
    ```
  </Accordion>
</AccordionGroup>

## Development with Docker

For development, use a modified compose file:

```yaml
version: '3.8'

services:
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile.dev
    volumes:
      - ./backend/src:/app/src
      - ello-data:/data
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=development

  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile.dev
    volumes:
      - ./frontend/src:/app/src
    ports:
      - "5173:5173"

volumes:
  ello-data:
```

This enables hot reload for both frontend and backend.
