# Local Development

This guide covers setting up Ello for local development on your machine.

## Prerequisites

- **Node.js 20+** - [Download from nodejs.org](https://nodejs.org/)
- **npm** - Comes with Node.js
- **Git** - For version control

## Clone the Repository

```bash
git clone https://github.com/smombartz/ello-claude.git
cd ello-claude
```

## Project Structure

```
ello-claude/
├── backend/           # Node.js/Fastify API server
│   ├── src/
│   │   ├── routes/    # API endpoint handlers
│   │   ├── services/  # Business logic
│   │   ├── schemas/   # TypeBox validation
│   │   └── types/     # TypeScript interfaces
│   ├── data/          # SQLite database and photos (created on first run)
│   └── package.json
├── frontend/          # React/Vite web application
│   ├── src/
│   │   ├── components/
│   │   ├── api/
│   │   └── lib/
│   └── package.json
├── mint.json          # Mintlify documentation config
└── README.md
```

## Backend Setup

<Steps>
  <Step title="Navigate to backend directory">
    ```bash
    cd backend
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```
    This installs Fastify, better-sqlite3, Sharp, and other dependencies.
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```
    The API server starts at `http://localhost:3000`
  </Step>
</Steps>

### Backend Environment Variables

Create a `.env` file in the backend directory (optional):

```bash
# Database location (default: ./data/contacts.db)
DATABASE_PATH=./data/contacts.db

# Photo storage location (default: ./data/photos)
PHOTOS_PATH=./data/photos

# Server port (default: 3000)
PORT=3000

# Environment (development or production)
NODE_ENV=development
```

### Backend Scripts

| Command | Description |
|---------|-------------|
| `npm run dev` | Start with hot reload (tsx watch) |
| `npm run build` | Compile TypeScript |
| `npm start` | Run compiled JavaScript |
| `npm test` | Run tests with Vitest |

## Vite Proxy Configuration

The frontend proxies API requests to the backend:

```typescript
// vite.config.ts
export default defineConfig({
  server: {
    proxy: {
      '/api': 'http://localhost:3000',
      '/photos': 'http://localhost:3000',
      '/health': 'http://localhost:3000'
    }
  }
});
```

## Database

The SQLite database is created automatically on first run at `backend/data/contacts.db`.

### Database Location

- **Development**: `backend/data/contacts.db`
- **Production**: Configured via `DATABASE_PATH` environment variable

### Reset Database

To start fresh, delete the data directory:

```bash
rm -rf backend/data
```

The database and directories will be recreated on next server start.

### View Database

Use any SQLite viewer to inspect the database:

```bash
# Using sqlite3 CLI
sqlite3 backend/data/contacts.db

# Common commands
.tables           # List all tables
.schema contacts  # Show table schema
SELECT COUNT(*) FROM contacts;  # Count contacts
```

## Photos

Contact photos are stored in `backend/data/photos/` organized by size:

```
data/photos/
├── thumbnail/    # 48x48px
├── small/        # 96x96px
├── medium/       # 200x200px
└── large/        # 400x400px
```

Each size directory uses a hash-based subdirectory structure:

```
thumbnail/
├── a1/
│   └── a1b2c3d4.jpg
├── b2/
│   └── b2c3d4e5.jpg
...
```

## Common Development Tasks

### Import Test Contacts

1. Start both backend and frontend
2. Go to http://localhost:5173
3. Click **Import**
4. Select a VCF file

### View API Responses

Use curl or a REST client:

```bash
# List contacts
curl http://localhost:3000/api/contacts

# Get contact detail
curl http://localhost:3000/api/contacts/1

# Health check
curl http://localhost:3000/health
```

### Inspect Logs

The backend uses Pino for logging. Enable pretty printing in development:

```bash
npm run dev | npx pino-pretty
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 3000 already in use">
    Kill the process using port 3000:
    ```bash
    lsof -i :3000
    kill -9 <PID>
    ```
    Or change the port via `PORT` environment variable.
  </Accordion>

  <Accordion title="Sharp installation fails">
    Sharp requires native dependencies. On macOS:
    ```bash
    brew install vips
    ```
    On Linux:
    ```bash
    apt-get install libvips-dev
    ```
  </Accordion>

  <Accordion title="Database locked">
    SQLite uses WAL mode which allows concurrent reads. If you see "database is locked" errors, ensure only one write operation runs at a time.
  </Accordion>

  <Accordion title="Frontend not connecting to backend">
    Ensure:
    1. Backend is running on port 3000
    2. Frontend Vite proxy is configured correctly
    3. No CORS errors in browser console
  </Accordion>
</AccordionGroup>
