# Yello Electron Desktop App Setup Guide

## Overview

Yello is now available as a native macOS desktop application. The Electron wrapper automatically:
- Spawns the backend Node.js server
- Manages the app lifecycle
- Serves the frontend from the backend
- Persists data in `~/Library/Application Support/Yello/`

## Quick Start (Development)

### Prerequisites
- Node.js 20+
- npm 10+
- macOS (for Electron building; Linux/Windows support can be added)

### Install & Run
```bash
# Install all dependencies
npm install
cd backend && npm install
cd ../frontend && npm install
cd ../electron && npm install

# Start Electron dev mode
npm run electron:dev
```

The app will:
1. Show splash screen
2. Spawn backend server (port 3456)
3. Poll health endpoint until ready
4. Open main window with dashboard

## Configuration

### Google OAuth Setup

1. **Create OAuth credentials** at [Google Cloud Console](https://console.cloud.google.com/)
   - Create new OAuth 2.0 Application
   - Type: Desktop Application
   - Authorized redirect URIs: `http://localhost:3456/api/auth/google/callback`

2. **Add credentials to Electron**
   - Copy `electron/.env.example` to `~/.config/Yello/.env`
   - Or place in `~/Library/Application Support/Yello/.env` (app data directory)
   - Fill in `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET`

Example `.env`:
```
GOOGLE_CLIENT_ID=your-client-id.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=your-client-secret
```

### Optional: Enrichment APIs

#### HERE.com Geocoding
```
HERE_API_KEY=your-api-key
```
Free tier: 250,000 requests/month

#### Apify LinkedIn Enrichment
```
APIFY_API_TOKEN=your-api-token
```

## Architecture

### Directory Structure
```
Electron main process (electron/src/main.ts)
├── Spawn backend
│   └── NODE: backend/dist/server.js
│       └── Fastify server (port 3456)
│           └── Serves frontend from frontend/dist/
├── Poll health endpoint
├── Create BrowserWindow
└── Load http://localhost:3456 in window
```

### Data Persistence
- **Auth DB**: `~/Library/Application Support/Yello/auth.db`
- **Per-user DBs**: `~/Library/Application Support/Yello/users/{userId}/contacts.db`
- **Photos**: `~/Library/Application Support/Yello/users/{userId}/photos/`
- **Session secret**: `~/Library/Application Support/Yello/.session-secret` (auto-generated)

### Environment Variables

**Electron passes these to backend:**
- `PORT=3456`
- `NODE_ENV=development|production`
- `AUTH_DATABASE_PATH=~/Library/Application Support/Yello/auth.db`
- `USER_DATA_PATH=~/Library/Application Support/Yello/users`
- `APP_URL=http://localhost:3456` (for OAuth redirects)
- `SESSION_SECRET=<auto-generated>` (persistent across app restarts)
- `GOOGLE_CLIENT_ID` (from .env)
- `GOOGLE_CLIENT_SECRET` (from .env)
- `HERE_API_KEY` (optional, from .env)
- `APIFY_API_TOKEN` (optional, from .env)

## Building for Distribution

### Prerequisites
- `icon.icns` in `build-resources/` (see `build-resources/ICON_README.md`)
- macOS Apple Developer ID (for notarization)

### Build DMG
```bash
npm run electron:build
```

Output: `electron/dist-electron/Yello-1.0.0-arm64.dmg` (universal Intel/Apple Silicon)

### Distribute
1. Sign and notarize the DMG with Apple Developer credentials
2. Distribute via website or App Store

## Development Workflows

### Starting Development
```bash
npm run electron:dev
```
- Frontend hot-reloads via Vite (port 5173)
- Backend hot-reloads via tsx (port 3456)
- Electron reloads main window on backend changes

### Running Backend Tests
```bash
cd backend && npm test
```

### Debugging Electron Main Process
The main process logs to console:
```
[Electron] Spawning backend...
[Electron] Waiting for backend...
[Electron] Backend is ready
```

Backend logs appear with `[Backend]` prefix.

### Stopping Dev Servers
```bash
kill $(lsof -ti :3456)  # Backend
kill $(lsof -ti :5173)  # Frontend (if running separate dev)
```

## Troubleshooting

### "Backend did not become ready"
- Check if port 3456 is already in use
- Verify `~/Library/Application Support/Yello/` is writable
- Check console logs for backend errors

### OAuth fails with "invalid_grant"
- Verify `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` in `.env`
- Ensure `http://localhost:3456/api/auth/google/callback` is in authorized redirect URIs
- Clear browser cookies and try again

### Data not persisting after restart
- Check `~/Library/Application Support/Yello/` exists
- Verify SQLite DBs created in correct location
- Check file permissions (should be 0o700 for app data dir)

### App icon is generic Electron icon
- Add `icon.icns` to `build-resources/` before building
- See `build-resources/ICON_README.md` for icon generation

## Security Notes

- Session secret is auto-generated and persisted per installation
- All OAuth tokens encrypted with SESSION_SECRET
- Data stored per-user with database isolation
- No external services contacted except:
  - Google OAuth (required for login)
  - HERE.com (optional, for geocoding)
  - Apify (optional, for enrichment)
