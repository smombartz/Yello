# vCard Parsing

Ello imports contacts from vCard (VCF) files. This document covers supported formats, field mappings, and edge case handling.

## Supported Versions

| Version | RFC | Support |
|---------|-----|---------|
| vCard 2.1 | (Legacy) | Full |
| vCard 3.0 | RFC 2426 | Full |
| vCard 4.0 | RFC 6350 | Full |

## vCard Structure

A vCard is a text file with a specific structure:

```
BEGIN:VCARD
VERSION:3.0
N:Smith;John;David;Mr.;Jr.
FN:Mr. John David Smith Jr.
ORG:Acme Corporation
TITLE:Chief Executive Officer
TEL;TYPE=mobile:+1 555 123 4567
TEL;TYPE=work:(555) 987-6543
EMAIL;TYPE=work:john@acme.com
EMAIL;TYPE=home:john.smith@gmail.com
ADR;TYPE=work:;;123 Main Street;New York;NY;10001;USA
BDAY:1985-03-15
NOTE:Met at conference 2024
PHOTO;ENCODING=b;TYPE=JPEG:/9j/4AAQSkZJRg...
END:VCARD
```

## Field Mappings

### Name Fields

| vCard Field | Ello Field | Description |
|-------------|------------|-------------|
| `N` | firstName, lastName | Structured name components |
| `FN` | displayName | Formatted display name (required) |

#### N Field Structure

```
N:Last;First;Middle;Prefix;Suffix
```

Example:
```
N:Smith;John;David;Mr.;Jr.
```

Parsed as:
- lastName: "Smith"
- firstName: "John"
- (middle, prefix, suffix stored in raw vCard)

#### FN Field Fallback

If `FN` is missing, Ello constructs it from `N`:

```typescript
displayName = [firstName, lastName].filter(Boolean).join(' ');
```

### Contact Methods

| vCard Field | Ello Field | Example |
|-------------|------------|---------|
| `TEL` | phones[] | `TEL;TYPE=mobile:+15551234567` |
| `EMAIL` | emails[] | `EMAIL;TYPE=work:john@example.com` |
| `ADR` | addresses[] | `ADR;TYPE=work:;;123 Main St;...` |

#### TEL Field

```
TEL;TYPE=mobile,voice:+1 555 123 4567
```

- **Normalization**: Converted to E.164 format using libphonenumber-js
- **Display**: Original formatting preserved in `phone_display`
- **Type**: Extracted from TYPE parameter

Type mapping:
| vCard Type | Ello Type |
|------------|-----------|
| `cell`, `mobile` | mobile |
| `work` | work |
| `home` | home |
| `fax` | fax |
| Other | other |

#### EMAIL Field

```
EMAIL;TYPE=work,pref:john@example.com
```

- **Primary**: Determined by `pref` type or first email
- **Type**: work, home, other

#### ADR Field

```
ADR;TYPE=work:PO Box;;123 Main Street;New York;NY;10001;USA
```

Structure:
```
ADR:PO Box;Extended;Street;City;State;Postal;Country
```

| vCard Field | Ello Field | Example |
|-------------|------------|---------|
| `ORG` | company | `ORG:Acme Corporation` |
| `TITLE` | title | `TITLE:Chief Executive Officer` |
| `NOTE` | notes | `NOTE:Met at conference` |
| `BDAY` | birthday | `BDAY:1985-03-15` |

### Extended Fields

| vCard Field | Ello Field | Example |
|-------------|------------|---------|
| `URL` | urls[] | `URL:https://example.com` |
| `X-SOCIALPROFILE` | socialProfiles[] | `X-SOCIALPROFILE;TYPE=twitter:johnsmith` |
| `IMPP` | instantMessages[] | `IMPP;TYPE=jabber:john@jabber.org` |
| `RELATED` | relatedPeople[] | `RELATED;TYPE=spouse:Jane Smith` |
| `CATEGORIES` | categories[] | `CATEGORIES:Work,VIP` |

#### X-SOCIALPROFILE

Apple-specific field for social profiles:

```
X-SOCIALPROFILE;TYPE=twitter;x-user=johnsmith:http://twitter.com/johnsmith
```

Parsed as:
- platform: "twitter"
- username: "johnsmith"
- url: "http://twitter.com/johnsmith"

#### IMPP

Instant messaging (RFC 4770):

```
IMPP;TYPE=work:xmpp:john@jabber.org
```

Parsed as:
- service: "xmpp"
- handle: "john@jabber.org"
- type: "work"

---

### Photo Field

```
PHOTO;ENCODING=b;TYPE=JPEG:/9j/4AAQSkZJRg...
```

or (vCard 4.0):

```
PHOTO:data:image/jpeg;base64,/9j/4AAQSkZJRg...
```

- **Encoding**: Base64 decoded
- **Processing**: Resized to 4 variants via Sharp
- **Storage**: Hash-based filename

## Encoding Handling

### Quoted-Printable

vCard 2.1/3.0 may use quoted-printable encoding:

```
NOTE;ENCODING=QUOTED-PRINTABLE:Meeting notes with =C3=BCber important details
```

Decoded: "Meeting notes with ĂĽber important details"

### Line Folding

Long lines are folded with leading whitespace:

```
NOTE:This is a very long note that continues
  on the next line with leading whitespace
  that indicates continuation.
```

Ello unfolds lines before parsing.

### Character Encoding

- **Expected**: UTF-8
- **Fallback**: Latin-1/ISO-8859-1
- **Detection**: Automatic based on content

## Multi-Contact Files

VCF files can contain multiple vCards:

```
BEGIN:VCARD
...contact 1...
END:VCARD
BEGIN:VCARD
...contact 2...
END:VCARD
```

Ello splits on `BEGIN:VCARD` and processes each independently.

---

## Error Handling

| Scenario | Handling |
|----------|----------|
| Missing `FN` | Construct from `N` field |
| Missing `N` and `FN` | Skip contact, log error |
| Invalid phone format | Store as-is, normalization fails gracefully |
| Invalid email format | Store as-is |
| Malformed vCard | Skip contact, log error |
| Invalid photo data | Import contact without photo |
| Unknown field | Preserve in `raw_vcard` |

## Parser Implementation

The parser uses a state machine approach:

```typescript
function parseVcf(content: string): ParseResult {
  // 1. Unfold lines (remove line continuations)
  const unfolded = unfoldLines(content);

  // 2. Split into vCard blocks
  const blocks = splitVCards(unfolded);

  // 3. Parse each block
  const contacts = [];
  const errors = [];

  for (let i = 0; i < blocks.length; i++) {
    try {
      const contact = parseSingleVCard(blocks[i]);
      contacts.push(contact);
    } catch (error) {
      errors.push({ line: i + 1, reason: error.message });
    }
  }

  return { contacts, errors };
}
```

## Testing vCard Parsing

### Test Cases

1. **Minimal vCard**: Only FN field
2. **Complete vCard**: All supported fields
3. **Multi-value fields**: Multiple emails, phones
4. **Encoded content**: Quoted-printable, base64
5. **International characters**: UTF-8, non-ASCII
6. **Malformed vCards**: Missing END, invalid structure
7. **Version differences**: 2.1, 3.0, 4.0 syntax
